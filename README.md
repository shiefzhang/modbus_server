# Modbus Command Server

一个用于声光设备的 Modbus 命令控制服务。程序同时启动 TCP 心跳服务和 FastAPI 接口服务：设备保持心跳连接，API 收到控制命令后复用该连接下发 Modbus 命令。

## 功能特性

- 监听设备心跳连接，默认端口 `10015`。
- 提供 HTTP API，默认端口 `8288`。
- 心跳连接只接收数据，不主动回复。
- 可识别设备直接返回的命令回显，避免把命令回显误统计为心跳。
- 默认发送命令给最近收到心跳的设备。
- 支持从心跳数据第 `4~11` 个字节提取设备标识。
- 支持通过连接 ID 或心跳设备标识指定设备发送命令。
- 自动计算并追加 Modbus CRC16。

## 环境要求

- Python 3.9+
- FastAPI
- Uvicorn
- Pydantic

安装依赖：

```bash
pip install fastapi uvicorn pydantic
```

## 启动服务

```bash
python modbus_api.py
```

启动后会同时运行：

| 服务 | 默认地址 | 说明 |
| --- | --- | --- |
| 心跳 TCP 服务 | `0.0.0.0:10015` | 设备连接并周期性发送心跳 |
| FastAPI 服务 | `0.0.0.0:8288` | 接收 HTTP 命令并下发 Modbus 帧 |

API 文档地址：

```text
http://127.0.0.1:8288/docs
```

## 设备标识

服务收到心跳数据后，会把心跳帧中的第 `4~11` 个字节作为设备标识保存。代码中对应：

```python
data[3:11].hex().upper()
```

例如心跳数据中第 `4~11` 个字节为：

```text
01 02 03 04 05 06 07 08
```

保存的设备标识就是：

```text
0102030405060708
```

设备标识会出现在：

- `GET /clients` 返回的 `device_identifier`
- `GET /stats` 返回的 `statistics.device_identifiers`
- `GET /heartbeat/status` 返回的 `last_device_identifier`

## API 用法

### 查看服务信息

```bash
curl http://127.0.0.1:8288/
```

### 查看在线设备

```bash
curl http://127.0.0.1:8288/clients
```

### 发送命令到最近收到心跳的设备

```bash
curl http://127.0.0.1:8288/send/1
```

POST 方式：

```bash
curl -X POST http://127.0.0.1:8288/command \
  -H "Content-Type: application/json" \
  -d '{"command":1}'
```

### 发送命令到指定设备

`client_id` 参数既可以使用连接 ID，例如 `device_0001`，也可以使用心跳里的设备标识。

```bash
curl http://127.0.0.1:8288/send/1/0102030405060708
```

POST 方式：

```bash
curl -X POST http://127.0.0.1:8288/command/0102030405060708 \
  -H "Content-Type: application/json" \
  -d '{"command":1}'
```

也可以在请求体中传入 `client_id`：

```bash
curl -X POST http://127.0.0.1:8288/command \
  -H "Content-Type: application/json" \
  -d '{"command":1,"client_id":"0102030405060708"}'
```

### 查看统计信息

```bash
curl http://127.0.0.1:8288/stats
```

### 查看心跳状态

```bash
curl http://127.0.0.1:8288/heartbeat/status
```

### 断开指定连接

该接口使用连接 ID，不使用心跳设备标识。

```bash
curl -X DELETE http://127.0.0.1:8288/client/device_0001
```

## 命令表

### 声光控制

| 命令 | 功能 | Modbus 数据区 |
| --- | --- | --- |
| `1` | 打开声音和灯光 | `01 06 04 0E 00 03` |
| `2` | 关闭声光 | `01 06 04 0E 00 00` |
| `5` | 单独打开声音 | `01 06 04 0E 00 01` |
| `6` | 单独打开灯光 | `01 06 04 0E 00 02` |

### 播放模式

| 命令 | 功能 | Modbus 数据区 |
| --- | --- | --- |
| `3` | 单曲循环模式 | `01 06 04 11 00 01` |
| `4` | 单曲模式 | `01 06 04 11 00 02` |

### 音调切换

命令 `11~40` 对应第一个文件夹中的第 `1~30` 条语音。

| 命令范围 | 功能 | Modbus 数据区规律 |
| --- | --- | --- |
| `11~40` | 切换到第 `1~30` 条语音 | `01 06 04 10 01 XX`，其中 `XX` 为 `01~1E` |

示例：

| 命令 | 功能 | Modbus 数据区 |
| --- | --- | --- |
| `11` | 第一条语音 | `01 06 04 10 01 01` |
| `30` | 第二十条语音 | `01 06 04 10 01 14` |
| `40` | 第三十条语音 | `01 06 04 10 01 1E` |

### 音量控制

| 命令 | 功能 | Modbus 数据区 |
| --- | --- | --- |
| `51` | 音量调为 1 级 | `01 06 04 0F 00 01` |
| `52` | 音量调为 5 级 | `01 06 04 0F 00 05` |
| `53` | 音量调为 10 级 | `01 06 04 0F 00 0A` |
| `54` | 音量调为 15 级 | `01 06 04 0F 00 0F` |
| `55` | 音量调为 20 级 | `01 06 04 0F 00 14` |
| `56` | 音量调为 25 级 | `01 06 04 0F 00 19` |
| `57` | 音量调为 30 级 | `01 06 04 0F 00 1E` |

## 工作流程

1. 设备连接 TCP 心跳服务 `10015`。
2. 设备定期发送心跳数据，服务端记录连接、心跳时间和设备标识。
3. HTTP API 收到命令。
4. 如果未指定设备，服务端选择最近收到心跳的设备。
5. 如果指定设备，服务端按连接 ID 或心跳设备标识查找在线连接。
6. 服务端生成 Modbus 帧、追加 CRC16，并通过设备连接发送。
7. 如果设备直接返回同一条命令帧，服务端记录为命令回显，不更新心跳时间和设备标识。

## 注意事项

- 心跳帧长度小于 11 字节时，设备标识会记录为 `null`。
- 默认发送接口要求至少有一个设备已经上报过心跳。
- 如果新连接上报了已有设备标识，服务会删除并关闭旧连接，由新连接接管该设备标识。
- 命令发送后不在发送线程里等待响应，避免和心跳接收线程抢读同一个 socket。
